🧱 Docker Image Layers & Build Cache

Each Dockerfile instruction becomes a cached layer. Rebuild after changing different things and watch which layers are reused (fast) vs rebuilt (slow).

🥞 Analogy: Layers are like a stack of pancakes made in order. Docker remembers each one. If you only change the top pancake, it keeps all the lower ones. But change a lower pancake and every pancake above it must be re-cooked — that’s why we put slow, rarely-changing steps (installing dependencies) below fast-changing ones (copying source code).
